▌一、AI工程化基础词汇（198个）

单词/词组 词性 AI领域释义
MLOpsn.机器学习工程，把机器学习从实验到生产的工程化流程和技术
Machine Learning Operationsn.机器学习工程，即MLOps
DevOpsn.开发运维一体化，MLOps的基础
DataOpsn.数据工程，MLOps的重要组成部分，处理数据的全流程
ModelOpsn.模型工程，MLOps的核心，处理模型的全生命周期管理
CI/CDn.持续集成/持续部署，MLOps的基础流程，自动化测试和部署
Continuous Integrationn.持续集成，代码提交后自动进行构建、测试
Continuous Deliveryn.持续交付，通过自动化流程把代码部署到生产环境
Continuous Trainingn.持续训练，MLOps特有的流程，当新数据到来时自动重新训练模型
CTn.持续训练，Continuous Training的缩写
experiment trackingn.实验跟踪，记录机器学习实验的各种信息，比如超参数、指标、代码版本、数据集版本等，方便复现和对比
model versioningn.模型版本管理，管理模型的不同版本，方便回滚和对比
data versioningn.数据版本管理，管理数据集的不同版本，保证训练的可复现性
feature storen.特征存储，集中管理特征的平台，实现特征的复用和共享，避免特征工程的重复工作
feature pipelinen.特征流水线，自动处理原始数据，生成特征的流程
training pipelinen.训练流水线，自动训练模型的流程，包括数据加载、预处理、训练、评估等步骤
inference pipelinen.推理流水线，自动处理推理请求的流程，包括数据预处理、模型推理、后处理等步骤
pipeline orchestrationn.流水线编排，管理流水线的运行、调度、依赖等
workflown.工作流，即pipeline
DAGn.有向无环图，用来表示流水线的任务和依赖关系
Directed Acyclic Graphn.有向无环图，即DAG
model registryn.模型注册中心，集中管理模型的存储、版本、元数据、生命周期状态等
model servingn.模型服务，把训练好的模型部署成服务，对外提供推理API
model monitoringn.模型监控，监控生产环境中模型的性能、数据漂移、服务可用性等
data driftn.数据漂移，生产环境中的输入数据分布和训练时的数据分布不一致的问题，会导致模型性能下降
concept driftn.概念漂移，输入和输出之间的关系发生变化的问题，也会导致模型性能下降
drift detectionn.漂移检测，检测数据漂移和概念漂移的技术
model retrainingn.模型重训练，当模型性能下降或者漂移发生时，用新数据重新训练模型
A/B testingn.A/B测试，同时上线两个版本的模型，对比它们的性能，选择更好的版本
canary deploymentn.金丝雀部署，先把新版本的模型上线给一小部分用户，观察没有问题再全量上线，降低风险
blue-green deploymentn.蓝绿部署，同时运行两个版本的服务，切换流量到新版本，有问题可以立刻切回旧版本，零 downtime
rollbackn./v.回滚，把模型或者服务切换回之前的版本
reproducibilityn.可复现性，机器学习实验能重复得到相同结果的能力，MLOps的核心目标之一
ML lifecyclen.机器学习生命周期，从问题定义、数据收集、特征工程、模型训练、评估、部署、监控、重训练的全流程
experimentationn.实验阶段，机器学习项目的初期阶段，尝试不同的模型、算法、超参数
productionn.生产环境，模型正式上线对外提供服务的环境
stagingn.预发环境，和生产环境配置一样，上线前测试用的环境
developmentn.开发环境，开发人员日常开发用的环境
testingn.测试环境，测试用的环境
environment isolationn.环境隔离，不同环境之间互相隔离，避免互相影响
containerizationn.容器化，把应用和依赖打包成容器，保证在不同环境中运行一致，MLOps的基础技术
Dockern.最常用的容器化工具
container imagen.容器镜像，打包好的应用和依赖
container registryn.容器镜像仓库，存储和管理容器镜像
orchestrationn.容器编排，管理大规模容器的部署、调度、扩缩容等
Kubernetesn.K8s，最常用的容器编排工具，现在是AI服务部署的标准基础设施
K8sn.Kubernetes的缩写
podn.K8s的最小调度单元，里面可以运行一个或者多个容器
deploymentn.K8s的资源类型，管理无状态应用的部署
servicen.K8s的资源类型，给应用提供稳定的访问入口
ingressn.K8s的资源类型，管理外部访问的路由
noden.K8s集群中的节点，即物理机或者虚拟机
clustern.集群，多个节点组成的集群
auto-scalingn.自动扩缩容，根据流量自动调整服务的实例数量，节约资源，保证性能
horizontal pod autoscalern.水平Pod扩缩容（HPA），K8s中根据CPU、内存或者自定义指标自动调整实例数量
HPAn.水平Pod扩缩容，Horizontal Pod Autoscaler的缩写
GPU schedulingn.GPU调度，K8s中调度GPU资源的能力，AI服务部署必备
nvidia-dockern.NVIDIA提供的Docker运行时，能让容器访问GPU资源
CUDAn.NVIDIA推出的GPU并行计算平台和编程模型，AI训练和推理必备
cuDNNn.NVIDIA推出的深度神经网络库，优化了常见的深度学习操作，加速训练和推理
TensorRTn.NVIDIA推出的深度学习推理优化引擎，能大幅提升模型推理速度，降低延迟
ONNXn.开放神经网络交换格式，不同框架训练的模型能转换成ONNX格式，然后在不同的推理引擎上运行
Open Neural Network Exchangen.开放神经网络交换格式，即ONNX
model optimizationn.模型优化，在几乎不损失精度的情况下，减少模型体积，提升推理速度，降低资源占用
quantizationn.量化，把模型的参数从32位浮点数（FP32）转换成更低精度的表示，比如FP16、INT8、INT4，减少内存占用，提升推理速度
post-training quantizationn.训练后量化（PTQ），模型训练完成后直接量化，不需要重新训练，简单方便
PTQn.训练后量化，Post-training Quantization的缩写
quantization-aware trainingn.量化感知训练（QAT），训练的时候就模拟量化的误差，让模型适应量化，精度损失更小
QATn.量化感知训练，Quantization-aware Training的缩写
FP32n.32位浮点数，模型训练常用的精度
FP16n.16位浮点数，混合精度训练和推理常用的精度，速度更快，内存占用更少
BF16n.16位脑浮点数，和FP16类似，但是动态范围更大，适合训练，Ampere及以上架构的GPU支持
INT8n.8位整数，推理常用的量化精度，速度快，内存占用少，精度损失小
INT4n.4位整数，大模型推理常用的量化精度，能大幅减少内存占用，让大模型能在消费级GPU上运行
AWQn. Activation-aware Weight Quantization，现在最流行的4位量化算法，精度损失很小，推理速度很快
GPTQn.大模型量化算法，也是常用的4位量化算法
SmoothQuantn.大模型量化算法，能实现无损INT8量化
pruningn.剪枝，删除模型中不重要的参数，减少模型体积和计算量
structured pruningn.结构化剪枝，删除整个通道、层等结构化的参数，能直接减少计算量，不需要特殊的推理引擎支持
unstructured pruningn.非结构化剪枝，删除单个不重要的参数，压缩率更高，但是需要特殊的推理引擎支持才能加速
distillationn.知识蒸馏，用大模型（教师模型）来训练小模型（学生模型），让小模型能达到接近大模型的性能
teacher modeln.教师模型，知识蒸馏中的大模型
student modeln.学生模型，知识蒸馏中的小模型
model compressionn.模型压缩，包含量化、剪枝、蒸馏等技术，减少模型体积，提升推理速度
