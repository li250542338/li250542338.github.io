一、函数基础知识点

1.1 什么是函数

函数是组织好的、可重复使用的、用来实现单一或相关联功能的代码段，能提高代码的复用性和模块化程度。

1.2 函数的定义与调用

def 函数名(参数列表):
    """函数文档字符串（可选）"""
    函数体
    return 返回值 # 可选

# 调用方法
函数名(实参列表)
💡 注意：函数必须先定义后调用，没有return语句时默认返回None

1.3 参数类型详解

参数类型 说明 示例
位置参数 调用时必须按定义顺序传入，数量必须匹配 def func(a, b): pass
默认参数 定义时给参数默认值，调用时可以不传 def func(a, b=10): pass
可变参数 接收任意多个位置参数，自动封装为元组 def func(*args): pass
关键字参数 接收任意多个键值对参数，自动封装为字典 def func(**kwargs): pass
命名关键字参数 限制关键字参数的名字，必须用参数名传入 def func(a, b, *, c, d): pass

1.4 参数传递顺序

混合使用时必须遵循以下顺序：

# 位置参数 → 默认参数 → 可变参数 → 命名关键字参数 → 关键字参数
def func(a, b=10, *args, c, d, **kwargs):
    pass

1.5 返回值

  • return可以返回多个值，自动封装为元组
  • 没有return或者return后面为空时，返回None
  • 执行到return语句时，函数立即终止执行
def calc(a, b):
    return a+b, a-b, a*b, a/b  # 返回4个值，接收时可以解包

add, sub, mul, div = calc(10, 5)

二、函数进阶知识点

2.1 变量作用域 & LEGB规则

Python查找变量的顺序：

  1. L（Local）：函数内部作用域
  2. E（Enclosing）：外部嵌套函数作用域（闭包常见）
  3. G（Global）：全局作用域
  4. B（Built-in）：Python内置作用域
🔍 关键字说明：
- global：在函数内部修改全局变量
- nonlocal：在嵌套函数中修改外层函数的变量 
count = 0  # 全局变量
def outer():
    num = 10  # 外层函数变量
    def inner():
        global count
        nonlocal num
        count += 1
        num += 1
        return count, num
    return inner

2.2 匿名函数（lambda）

适合简单的、一次性使用的小函数：

# 语法：lambda 参数列表: 表达式
add = lambda x, y: x + y
print(add(3, 5))  # 输出8

# 常见用法：配合sort、map等函数
students = [("张三", 18), ("李四", 16), ("王五", 20)]
students.sort(key=lambda x: x[1])  # 按年龄排序

2.3 高阶函数

接收函数作为参数，或者返回函数的函数：

① map() 映射

nums = [1, 2, 3, 4]
result = map(lambda x: x**2, nums)
print(list(result))  # 输出 [1, 4, 9, 16]

② filter() 过滤

nums = [1, 2, 3, 4, 5, 6]
result = filter(lambda x: x%2 ==0, nums)
print(list(result))  # 输出 [2, 4, 6]

③ reduce() 归约

from functools import reduce
nums = [1, 2, 3, 4]
result = reduce(lambda x, y: x+y, nums)
print(result)  # 输出 1+2+3+4=10

2.4 闭包

内层函数引用了外层函数的变量，并且外层函数返回内层函数，就形成了闭包：

def make_counter():
    count = 0
    def counter():
        nonlocal count
        count += 1
        return count
    return counter

c = make_counter()
print(c())  # 1
print(c())  # 2
print(c())  # 3

2.5 装饰器

在不修改原函数代码的前提下，给函数增加额外功能，遵循开闭原则：

import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"函数{func.__name__}执行耗时：{end-start}s")
        return result
    return wrapper

# 使用装饰器
@timer
def work():
    time.sleep(1)
    print("工作完成")

work()  # 自动打印执行时间

2.6 递归函数

函数自己调用自己，必须有终止条件：

def factorial(n):
    if n == 1:  # 终止条件
        return 1
    return n * factorial(n-1)

print(factorial(5))  # 5! = 120
⚠️ 注意：Python默认递归深度限制为1000层，超过会抛出RecursionError

三、难点 & 易错点（90%的人都踩过的坑）

3.1 参数传递机制的坑

Python参数传递是「对象引用传递」，分两种情况：

  • 不可变对象（数字、字符串、元组）：传递的是值的副本，函数内部修改不影响外部
  • 可变对象（列表、字典、集合）：传递的是对象本身的引用，函数内部修改会影响外部
def func(a, b):
    a += 10
    b.append(4)

x = 5
y = [1,2,3]
func(x, y)
print(x)  # 输出5，不变
print(y)  # 输出[1,2,3,4]，被修改了！
❌ 经典坑：默认参数用可变对象 
def func(a, b=[]):
    b.append(a)
    return b

print(func(1))  # [1] 正常
print(func(2))  # [1,2] 异常！默认参数被复用了
print(func(3))  # [1,2,3] 越来越大

✅ 正确写法：默认参数用None代替可变对象

def func(a, b=None):
    if b is None:
        b = []
    b.append(a)
    return b

3.2 作用域查找的坑

count = 10
def func():
    print(count)  # 报错！UnboundLocalError
    count = 20

func()

❓ 原因：Python在编译函数时，发现有给count赋值的操作，就会把count当成局部变量，执行print时局部变量还没赋值，所以报错。

✅ 解决：用global关键字声明是全局变量。

3.3 闭包的延迟绑定坑

def create_func():
    funcs = []
    for i in range(3):
        def func():
            return i*i
        funcs.append(func)
    return funcs

f1, f2, f3 = create_func()
print(f1())  # 期望0，实际输出4！
print(f2())  # 期望1，实际输出4！
print(f3())  # 期望4，实际输出4！

❓ 原因：闭包中的变量i是延迟绑定的，函数调用时才会找i的值，此时循环已经结束，i=2，所以结果都是4。

✅ 解决：用默认参数在定义时就绑定值

def create_func():
    funcs = []
    for i in range(3):
        def func(i=i):  # 用默认参数绑定当前i的值
            return i*i
        funcs.append(func)
    return funcs

3.4 装饰器的坑

用了装饰器之后，原函数的元信息（__name__、__doc__等）会被替换成装饰器内部函数的信息：

@timer
def work():
    """工作函数"""
    pass

print(work.__name__)  # 输出'wrapper'，而不是'work'
print(work.__doc__)  # 输出None，丢失了原函数的文档

✅ 解决：用functools.wraps保留原函数元信息

import functools
def timer(func):
    @functools.wraps(func)  # 保留原函数信息
    def wrapper(*args, **kwargs):
        # ... 装饰器逻辑
    return wrapper

3.5 *args和**kwargs的顺序坑

必须先写*args，再写**kwargs，顺序错了直接报语法错误！

def func(*args, **kwargs):  ✅ 正确
    pass

def func(**kwargs, *args):  ❌ 错误，语法报错
    pass

四、函数最佳实践

  1. 单一职责原则：一个函数只做一件事，代码不超过50行最佳
  2. 命名规范：见名知意，用小写字母加下划线，比如get_user_info
  3. 参数不宜过多：参数超过5个时考虑用数据类或者关键字参数
  4. 合理添加文档字符串：说明函数功能、参数、返回值、异常等
  5. 避免修改传入的可变参数：如果需要修改，先创建副本再操作
  6. 递归深度控制：超过100层的递归考虑用循环代替，避免栈溢出
  7. 合理使用装饰器：不要过度使用，避免代码逻辑过于复杂难以调试
