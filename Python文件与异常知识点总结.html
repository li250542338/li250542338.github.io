二、知识点详细讲解

2.1 文件操作

2.1.1 打开和关闭文件

Python使用open()函数打开文件，使用完后必须关闭文件释放资源

# 普通打开方式（容易忘记关闭文件）
f = open('test.txt', 'r', encoding='utf-8')
content = f.read()
f.close()

# 推荐方式：使用with关键字，会自动关闭文件
with open('test.txt', 'r', encoding='utf-8') as f:
    content = f.read()

参数说明：

  • 第一个参数：文件路径（相对路径或绝对路径）
  • 第二个参数：打开模式
    • 'r'：只读模式（默认）
    • 'w'：写入模式（文件不存在则创建，存在则覆盖原有内容）
    • 'a'：追加模式（文件不存在则创建，存在则在末尾追加内容）
    • 'r+'：读写模式
    • 'b'：二进制模式（比如读写图片、视频等非文本文件，和其他模式组合使用：'rb'、'wb'）
  • encoding='utf-8'：指定文件编码，避免中文乱码问题

2.1.2 读取文件的三种方式

# 1. read()：一次性读取整个文件内容，返回字符串
with open('test.txt', 'r', encoding='utf-8') as f:
    content = f.read()
    print(content)

# 2. readlines()：按行读取所有内容，返回列表，每个元素是一行内容
with open('test.txt', 'r', encoding='utf-8') as f:
    lines = f.readlines()
    for line in lines:
        print(line.strip())  # strip()去掉每行末尾的换行符

# 3. 逐行读取（适合大文件，不会一次性占用太多内存）
with open('test.txt', 'r', encoding='utf-8') as f:
    for line in f:
        print(line.strip())

2.1.3 写入文件

# 覆盖写入（原有内容会被清空）
with open('test.txt', 'w', encoding='utf-8') as f:
    f.write("第一行内容\n")
    f.write("第二行内容\n")  # 手动加换行符实现多行写入

# 追加写入（在文件末尾添加内容，原有内容保留）
with open('test.txt', 'a', encoding='utf-8') as f:
    f.write("第三行追加内容\n")

2.1.4 文件路径

  • 相对路径：相对于当前运行程序的目录 
    # 当前目录下的test.txt
open('test.txt', 'r')
# 当前目录下data文件夹里的test.txt
open('data/test.txt', 'r')
# 上一级目录里的test.txt
open('../test.txt', 'r')
  • 绝对路径：从磁盘根目录开始的完整路径 
    # Windows系统
open('C:/Users/xxx/Documents/test.txt', 'r')
# macOS/Linux系统
open('/home/xxx/Documents/test.txt', 'r')

2.2 异常处理

异常是程序运行时出现的错误，如果不处理，程序会直接崩溃退出，异常处理可以让程序优雅地处理错误，继续运行。

2.2.1 基本语法：try-except

try:
    # 可能出现错误的代码
    num = int(input("请输入一个数字："))
    result = 10 / num
except ZeroDivisionError:
    # 捕获到除零错误时执行
    print("错误：不能除以0！")
except ValueError:
    # 捕获到值错误（输入不是数字）时执行
    print("错误：请输入有效的数字！")
else:
    # 没有出现异常时执行
    print(f"计算结果是：{result}")
finally:
    # 不管有没有异常都会执行，通常用来释放资源
    print("程序执行结束")

2.2.2 常见异常类型

异常类型 说明
FileNotFoundError 文件不存在错误
ZeroDivisionError 除零错误
ValueError 值错误（比如把字符串转数字失败）
TypeError 类型错误（比如字符串和数字相加）
IndexError 索引错误（列表访问超出范围）
KeyError 键错误（字典访问不存在的键）
PermissionError 权限错误（没有权限读写文件）

2.2.3 pass关键字

捕获异常后什么都不做，静默处理

try:
    with open('不存在的文件.txt', 'r') as f:
        content = f.read()
except FileNotFoundError:
    pass  # 文件不存在就跳过，不做任何处理

2.3 数据持久化：json模块

JSON是一种轻量级的数据交换格式，Python的json模块可以把Python数据结构（字典、列表等）序列化保存到文件，也可以从文件中反序列化读取回来。

import json

# 1. 写入json文件：json.dump()
user_info = {
    "name": "张三",
    "age": 18,
    "hobbies": ["读书", "编程", "跑步"]
}
with open('user.json', 'w', encoding='utf-8') as f:
    # ensure_ascii=False保证中文正常保存，indent=4格式化输出更美观
    json.dump(user_info, f, ensure_ascii=False, indent=4)

# 2. 读取json文件：json.load()
with open('user.json', 'r', encoding='utf-8') as f:
    user_info = json.load(f)
    print(user_info["name"])  # 输出：张三

  1. with关键字的使用：必须掌握，是Python中操作文件的标准方式，避免资源泄漏
  2. 文件的读写操作：掌握常见的读取方式和写入方式，区分覆盖和追加模式

  5. 常见异常类型的识别：能够根据报错信息快速定位错误类型

四、本章难点

  1. 文件路径的处理：相对路径和绝对路径的区分，不同操作系统路径格式的差异，路径拼接的问题
  2. 异常的合理使用：知道什么时候需要捕获异常，不要过度捕获异常，也不要什么异常都不处理
  3. 二进制文件的读写：图片、视频等非文本文件的读写操作，需要使用'b'模式
  4. 多异常的捕获顺序：子类异常要放在父类异常前面捕获，否则子类异常会被父类异常拦截
  5. JSON序列化的限制：不是所有Python对象都能序列化为JSON，比如自定义类的实例默认不能直接序列化

五、高频易错点汇总

1. ❌ 忘记关闭文件

# 错误：如果read()报错，close()不会执行，文件资源泄漏
f = open('test.txt', 'r')
content = f.read()
f.close()

# 正确：使用with关键字，自动关闭文件
with open('test.txt', 'r') as f:
    content = f.read()

2. ❌ 中文文件读写乱码

# 错误：不指定encoding参数，Windows系统默认编码是gbk，会导致中文乱码
with open('test.txt', 'w') as f:
    f.write("中文内容")

# 正确：指定encoding='utf-8'
with open('test.txt', 'w', encoding='utf-8') as f:
    f.write("中文内容")

3. ❌ 读取文件后指针位置问题

# 错误：read()读取完整个文件后，文件指针在末尾，再次read()会得到空字符串
with open('test.txt', 'r') as f:
    content1 = f.read()
    content2 = f.read()  # 内容为空

# 正确：需要重新定位指针到开头
with open('test.txt', 'r') as f:
    content1 = f.read()
    f.seek(0)  # 指针移动到开头
    content2 = f.read()

4. ❌ 写入文件时忘记加换行符

# 错误：所有内容会写在同一行
with open('test.txt', 'w') as f:
    f.write("第一行")
    f.write("第二行")

# 正确：每行末尾加\n
with open('test.txt', 'w') as f:
    f.write("第一行\n")
    f.write("第二行\n")

5. ❌ 捕获异常太宽泛

# 错误：捕获所有异常，会隐藏真正的错误，不推荐
try:
    # 代码
except:
    print("出错了")

# 正确：捕获具体的异常类型，只处理你预期的错误
try:
    # 代码
except FileNotFoundError:
    print("文件不存在")

6. ❌ JSON序列化中文乱码

# 错误：不指定ensure_ascii=False，中文会被转成Unicode编码
json.dump(user_info, f)

# 正确：指定ensure_ascii=False
json.dump(user_info, f, ensure_ascii=False, indent=4)

六、实战案例：用户登录注册系统

import json

# 读取用户数据
def load_users():
    try:
        with open('users.json', 'r', encoding='utf-8') as f:
            return json.load(f)
    except FileNotFoundError:
        return {}

# 保存用户数据
def save_users(users):
    with open('users.json', 'w', encoding='utf-8') as f:
        json.dump(users, f, ensure_ascii=False, indent=4)

# 用户注册
def register():
    users = load_users()
    username = input("请输入用户名：")
    if username in users:
        print("用户名已存在！")
        return
    password = input("请输入密码：")
    users[username] = {"password": password}
    save_users(users)
    print("注册成功！")

# 用户登录
def login():
    users = load_users()
    username = input("请输入用户名：")
    password = input("请输入密码：")
    if username in users and users[username]["password"] == password:
        print("登录成功！")
    else:
        print("用户名或密码错误！")

# 主程序
while True:
    print("\n1. 注册\n2. 登录\n3. 退出")
    choice = input("请输入你的选择：")
    if choice == "1":
        register()
    elif choice == "2":
        login()
    elif choice == "3":
        break
    else:
        print("无效输入，请重新输入！")
