🎯 贯穿实例：校园人员管理系统
本次所有知识点都会围绕「校园人员管理系统」展开，我们会实现包含学生、老师两类人员的完整管理逻辑，所有语法点都搭配可运行的代码示例，边学边练更易懂！

一、核心知识点总结

1. 基础概念

类是创建对象的模板，对象是类的具体实例，就像「设计图纸」和「实际盖好的房子」的关系。

  • 定义规范：类名使用大驼峰命名法
  • 实例化：根据类创建具体对象的过程
# 定义基础的人员类（图纸）
class Person:
    # 构造方法：实例化时自动执行，用来初始化属性
    def __init__(self, name, age):
        self.name = name  # 姓名属性
        self.age = age    # 年龄属性

# 实例化创建两个具体的人员对象（实际的房子）
student1 = Person("张三", 18)
teacher1 = Person("李老师", 35)

print(f"学生：{student1.name}，年龄{student1.age}")
print(f"老师：{teacher1.name}，年龄{teacher1.age}")
运行输出：
学生：张三，年龄18
老师：李老师，年龄35

2. 属性分类

我们给Person类增加更多属性，区分不同属性类型：

属性类型 定义方式 特点 实例对应
实例属性 在__init__中通过self.xxx定义 每个实例独有，互不影响 每个人的姓名、年龄都不一样
类属性 直接在类内部、方法外定义 所有实例共享，属于类本身 所有人员所属的学校名称是同一个
私有属性 以双下划线__开头命名 外部无法直接访问，会被自动重命名 每个人的身份证号是隐私，不能随便修改
保护属性 以单下划线_开头命名 约定俗成的私有属性，建议不要直接外部访问 人员的内部编号，仅系统内部使用 
class Person:
    # 类属性：所有人员共享的学校名称
    school_name = "XX市第一中学"
    # 类属性：统计总人数
    total_count = 0

    def __init__(self, name, age, id_card, _inner_id):
        self.name = name                # 普通实例属性
        self.age = age                  # 普通实例属性
        self.__id_card = id_card        # 私有属性：身份证号
        self._inner_id = _inner_id      # 保护属性：内部编号
        # 每创建一个实例，总人数+1
        Person.total_count += 1

# 实例化两个对象
p1 = Person("张三", 18, "1101012008xxxxxxx", "S001")
p2 = Person("李老师", 35, "1101011991xxxxxxx", "T001")

# 类属性所有实例共享
print(p1.school_name)
print(p2.school_name)
print(Person.total_count)

# 私有属性外部无法直接访问
# print(p1.__id_card)  # 报错！AttributeError: 'Person' object has no attribute '__id_card'
运行输出：
XX市第一中学
XX市第一中学
2

3. 方法分类

给Person类增加不同类型的方法，实现管理功能：

方法类型 装饰器 默认第一个参数 调用方式 实例对应
实例方法 无 self（代表当前实例） 只能通过实例调用 每个人的自我介绍，需要访问自己的姓名
类方法 @classmethod cls（代表当前类） 可以通过类或实例调用 统计学校总人数，需要访问类属性total_count
静态方法 @staticmethod 无 可以通过类或实例调用 工具方法，比如判断年龄是否符合入学要求，不需要访问实例/类属性
魔术方法 前后双下划线命名（如__init__） 按规则定义 自动触发调用 打印实例时自动输出友好的信息 
class Person:
    school_name = "XX市第一中学"
    total_count = 0

    def __init__(self, name, age, id_card, _inner_id):
        self.name = name
        self.age = age
        self.__id_card = id_card
        self._inner_id = _inner_id
        Person.total_count += 1

    # 实例方法：自我介绍
    def introduce(self):
        print(f"大家好，我叫{self.name}，今年{self.age}岁，来自{self.school_name}")

    # 类方法：获取总人数
    @classmethod
    def get_total_count(cls):
        print(f"学校当前总人数：{cls.total_count}人")

    # 静态方法：判断年龄是否符合入学要求（年龄>=6岁）
    @staticmethod
    def is_age_valid(age):
        return age >= 6

    # 魔术方法：打印实例时自动调用
    def __str__(self):
        return f"Person(name={self.name}, age={self.age}, id={self._inner_id})"

# 测试方法
p1 = Person("张三", 18, "1101012008xxxxxxx", "S001")
p1.introduce()

Person.get_total_count()

print(Person.is_age_valid(5))
print(Person.is_age_valid(6))

print(p1)
运行输出：
大家好，我叫张三，今年18岁，来自XX市第一中学
学校当前总人数：1人
False
True
Person(name=张三, age=18, id=S001)

4. 面向对象三大特性

我们基于Person类扩展学生和老师类，演示三大特性：

① 封装

把属性和方法封装到类内部，对外仅暴露必要的访问接口，隐藏内部实现细节。比如我们的身份证号是私有属性，外部只能通过我们提供的get_id_card()方法访问，避免直接修改导致的错误：

class Person:
    def __init__(self, name, id_card):
        self.name = name
        self.__id_card = id_card
    
    # 对外提供访问私有属性的接口
    def get_id_card(self):
        # 可以在这里加权限校验，比如只有管理员才能查看完整身份证号
        return self.__id_card[:6] + "********" + self.__id_card[-4:]

p = Person("张三", "110101200801011234")
print(p.get_id_card())
运行输出：110101********1234，隐藏了中间生日部分

② 继承

子类可以继承父类所有非私有属性和方法，减少重复代码。我们创建Student类和Teacher类继承Person类，只需要实现自己特有的功能：

# 学生类继承Person类
class Student(Person):
    def __init__(self, name, age, id_card, _inner_id, student_id, grade):
        # 调用父类的构造方法，继承父类的属性
        super().__init__(name, age, id_card, _inner_id)
        # 子类特有的属性
        self.student_id = student_id  # 学号
        self.grade = grade            # 年级

    # 子类特有的方法：交作业
    def submit_homework(self, subject):
        print(f"{self.name}提交了{subject}作业")

# 老师类继承Person类
class Teacher(Person):
    def __init__(self, name, age, id_card, _inner_id, teacher_id, subject):
        super().__init__(name, age, id_card, _inner_id)
        self.teacher_id = teacher_id  # 工号
        self.subject = subject        # 授课科目

    # 子类特有的方法：布置作业
    def assign_homework(self):
        print(f"{self.subject}李老师布置了今天的作业")

# 测试继承
s1 = Student("张三", 18, "1101012008xxxxxxx", "S001", "2023001", "高三")
s1.introduce()  # 直接继承了父类的introduce方法，不需要重复写
s1.submit_homework("数学")

t1 = Teacher("李老师", 35, "1101011991xxxxxxx", "T001", "T0001", "数学")
t1.introduce()
t1.assign_homework()
运行输出：
大家好，我叫张三，今年18岁，来自XX市第一中学
张三提交了数学作业
大家好，我叫李老师，今年35岁，来自XX市第一中学
数学李老师布置了今天的作业

③ 多态

同一个方法在不同子类中有不同的实现，无需修改调用方代码即可适配不同类型。比如我们重写学生和老师的introduce()方法：

class Student(Person):
    # 重写父类的introduce方法
    def introduce(self):
        print(f"大家好，我是{self.grade}的学生{self.name}，我的学号是{self.student_id}")

class Teacher(Person):
    # 重写父类的introduce方法
    def introduce(self):
        print(f"大家好，我是教{self.subject}的{self.name}老师，我的工号是{self.teacher_id}")

# 多态演示：同一个welcome函数，传不同的实例会有不同的效果
def welcome(person):
    person.introduce()

s1 = Student("张三", 18, "1101012008xxxxxxx", "S001", "2023001", "高三")
t1 = Teacher("李老师", 35, "1101011991xxxxxxx", "T001", "T0001", "数学")

welcome(s1)
welcome(t1)
运行输出：
大家好，我是高三的学生张三，我的学号是2023001
大家好，我是教数学的李老师老师，我的工号是T0001
💡 多态的好处：不需要修改welcome函数，新增任何Person的子类都能直接适配，扩展性极强！

5. 进阶知识点

① @property装饰器

把方法转换为属性调用，实现属性的getter、setter逻辑，控制属性的访问和修改规则。比如我们给学生加成绩属性，限制分数只能在0-100之间：

class Student(Person):
    def __init__(self, name, age, id_card, _inner_id, student_id, grade):
        super().__init__(name, age, id_card, _inner_id)
        self.student_id = student_id
        self.grade = grade
        self._score = 0  # 私有成绩属性

    @property
    def score(self):
        """获取成绩"""
        return self._score

    @score.setter
    def score(self, value):
        """设置成绩，加校验逻辑"""
        if 0 <= value <= 100:
            self._score = value
        else:
            raise ValueError("成绩必须在0-100之间！")

s = Student("张三", 18, "1101012008xxxxxxx", "S001", "2023001", "高三")
s.score = 95  # 像访问属性一样设置值
print(s.score)

s.score = 120  # 报错：ValueError: 成绩必须在0-100之间！
运行输出：95
然后抛出错误：ValueError: 成绩必须在0-100之间！

② 抽象类

通过abc模块实现，不能被实例化，要求子类必须实现指定的抽象方法，用来规范接口定义。比如我们要求所有人员都必须实现work()方法：

from abc import ABC, abstractmethod

class Person(ABC):
    @abstractmethod
    def work(self):
        """抽象方法，子类必须实现"""
        pass

class Student(Person):
    def work(self):
        print("学生的工作是好好学习")

class Teacher(Person):
    def work(self):
        print("老师的工作是教书育人")

# p = Person()  # 报错！抽象类不能被实例化
s = Student()
s.work()
运行输出：学生的工作是好好学习

二、常见难点解析

1. super()函数的正确使用

多继承场景下，super()不是简单调用直接父类，而是按照MRO（方法解析顺序）查找下一个类：

class A:
    def hello(self):
        print("Hello from A")

class B(A):
    def hello(self):
        print("Hello from B")
        super().hello()

class C(A):
    def hello(self):
        print("Hello from C")
        super().hello()

class D(B, C):
    def hello(self):
        print("Hello from D")
        super().hello()

d = D()
d.hello()

# 查看MRO顺序
print(D.__mro__)
运行输出：
Hello from D
Hello from B
Hello from C
Hello from A
MRO顺序：(<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

2. 深浅拷贝与类实例

当类属性包含列表、字典等引用类型时，浅拷贝和深拷贝区别很大：

import copy

class Student:
    def __init__(self, name, courses):
        self.name = name
        self.courses = courses  # courses是列表，引用类型

s1 = Student("张三", ["数学", "语文"])
s2 = copy.copy(s1)  # 浅拷贝
s3 = copy.deepcopy(s1)  # 深拷贝

s1.courses.append("英语")
print(s1.courses)
print(s2.courses)
print(s3.courses)
运行输出：
['数学', '语文', '英语']
['数学', '语文', '英语'] → 浅拷贝共享引用类型
['数学', '语文'] → 深拷贝完全独立

三、高频易错点汇总（结合实例）

1. 类属性修改的坑

❌ 错误写法：用实例修改类属性，只会给实例新增属性，不会修改类本身

class Person:
    total_count = 0

p1 = Person()
p1.total_count = 1  # 这不是修改类属性，是给p1新增了一个实例属性total_count
print(Person.total_count)  # 输出还是0！
print(p1.total_count)  # 输出1，访问的是自己的实例属性

✅ 正确做法：修改类属性必须通过类名访问

Person.total_count = 1
print(Person.total_count)  # 输出1

2. 可变默认参数的坑

❌ 错误写法：用列表作为默认参数，会被所有实例共享

class Student:
    def __init__(self, name, courses=[]):
        self.name = name
        self.courses = courses

s1 = Student("张三")
s1.courses.append("数学")
s2 = Student("李四")
print(s2.courses)  # 输出['数学']！李四没选课也有数学课程，被共享了

✅ 正确做法：用None作为默认值，内部初始化列表

class Student:
    def __init__(self, name, courses=None):
        self.name = name
        self.courses = courses if courses is not None else []

3. 实例方法忘记加self参数

❌ 错误写法：

class Person:
    def introduce():  # 忘记加self
        print("你好")

p = Person()
p.introduce()  # 报错！调用时会自动传入self参数，方法没有接收位

✅ 正确做法：实例方法第一个参数必须是self

def introduce(self):
    print("你好")

4. 重写父类方法参数不匹配

父类方法定义为def work(self, hours)，子类重写为def work(self)，调用时会导致参数不匹配错误：

class Person:
    def work(self, hours):
        print(f"工作了{hours}小时")

class Student(Person):
    def work(self):  # 参数和父类不匹配
        print("好好学习")

s = Student()
s.work(8)  # 报错！work() takes 1 positional argument but 2 were given

✅ 正确做法：重写时保持参数一致，或者用*args, **kwargs做兼容

def work(self, *args, **kwargs):
    print("好好学习")
