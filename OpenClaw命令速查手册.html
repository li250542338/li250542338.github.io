🚀 快速入门 - 最常用的命令

命令 作用 使用场景
openclaw status 查看系统状态 检查OpenClaw是否正常运行
openclaw doctor 健康检查和修复 遇到问题时诊断和修复
openclaw gateway start 启动Gateway服务 开机后启动服务
openclaw gateway stop 停止Gateway服务 需要重启服务时
openclaw gateway restart 重启Gateway服务 配置更改后应用
openclaw logs --follow 实时查看日志 调试问题时查看日志
openclaw dashboard 打开Web控制面板 通过浏览器管理
openclaw tui 打开终端UI 在终端中对话

🔧 安装配置相关命令

openclaw setup
初始化配置和工作区
常用选项：
--workspace <dir> - 指定工作区路径（默认 ~/.openclaw/workspace）
--wizard - 运行向导式配置
--non-interactive - 非交互式配置
openclaw onboard
交互式初始化向导（Gateway、工作区、技能）
常用选项：
--reset - 重置配置、凭证和会话
--mode <local|remote> - 选择本地或远程模式
openclaw configure
交互式配置向导（模型、通道、技能、Gateway）
常用选项：
--section <section> - 只配置特定部分
openclaw config
非交互式配置管理
config get <path>
读取配置值（使用点号路径，如 gateway.port）
config set <path> <value>
设置配置值
config unset <path>
删除配置值
config file
显示配置文件路径
config schema
显示配置JSON Schema
config validate
验证配置是否有效
openclaw completion
生成Shell自动补全脚本
常用选项：
--shell <zsh|bash|powershell|fish> - 指定Shell类型
--install - 安装到Shell配置文件

🏠 Gateway服务管理

⚠️ Gateway是OpenClaw的核心服务！
几乎所有功能都需要Gateway运行才能使用。
openclaw gateway
管理Gateway服务（启动、停止、重启、状态检查）
gateway status
检查Gateway服务状态
gateway start
启动Gateway服务
gateway stop
停止Gateway服务
gateway restart
重启Gateway服务（配置更改后使用）
gateway install
安装Gateway为系统服务（开机自启）
gateway uninstall
卸载Gateway系统服务
openclaw daemon
Gateway服务的旧别名（功能同上）
daemon status / daemon start / daemon stop / daemon restart / daemon install / daemon uninstall
openclaw logs
查看Gateway日志
常用选项：
--follow - 实时跟踪日志（类似 tail -f）
--limit <n> - 只显示最后n行
--local-time - 显示本地时间
--json - JSON格式输出
openclaw dashboard
打开Web控制面板
在浏览器中打开OpenClaw的Web管理界面

🤖 Agent管理

openclaw agent
运行单次Agent对话
必需参数：
--message <text> 或 -m <text> - 发送的消息
会话选择（必需一个）：
--to <dest> - 目标会话
--session-id <id> - 会话ID
--agent <id> - Agent ID
openclaw agents
管理独立Agent（工作区、认证、路由）
agents list
列出所有配置的Agent
agents add [name]
添加新的独立Agent
agents delete <id>
删除Agent及其工作区和状态
agents bindings
列出路由绑定
agents bind
添加Agent的路由绑定
agents unbind
移除Agent的路由绑定
openclaw sessions
管理对话会话
常用选项：
--active <minutes> - 显示最近活跃的会话
--agent <id> - 按Agent过滤
--all-agents - 显示所有Agent的会话
sessions cleanup
清理过期或孤立的会话

🔨 系统维护

openclaw status
显示会话健康状态和最近联系人
常用选项：
--deep - 深度健康探测
--usage - 显示模型提供商使用情况
--all - 完整诊断
openclaw health
获取Gateway运行健康状态
常用选项：
--verbose - 强制实时探测并显示连接详情
openclaw doctor
健康检查和快速修复
常用选项：
--repair 或 --fix - 尝试自动修复检测到的问题
--deep - 扫描系统服务查找额外的Gateway安装
--yes - 接受默认值不提示
openclaw backup
创建和验证OpenClaw状态的本地备份
backup create
创建备份归档
backup verify <archive>
验证备份归档
openclaw update
更新已安装的CLI
update status
检查更新状态
update wizard
运行更新向导
openclaw reset
重置本地配置/状态（保留CLI安装）
常用选项：
--scope <config|config+creds+sessions|full> - 重置范围
--yes - 确认不提示
--dry-run - 预览不执行
openclaw uninstall
卸载Gateway服务和本地数据（保留CLI）
常用选项：
--service - 仅卸载服务
--state - 仅卸载状态
--workspace - 仅卸载工作区
--all - 全部卸载
--yes - 确认不提示
--dry-run - 预览不执行

💡 实用技巧与故障排除

🔧 常见问题排查流程

  1. 第一步： 运行 openclaw status 查看基本状态
  2. 第二步： 运行 openclaw doctor --repair 自动修复
  3. 第三步： 查看日志 openclaw logs --follow
  4. 第四步： 重启Gateway openclaw gateway restart

⚠️ 全局标志（所有命令通用）

标志 作用
--dev 在 ~/.openclaw-dev 下隔离状态
--profile <name> 使用指定配置文件
--no-color 禁用颜色输出
--json JSON格式输出
--version / -v 显示版本并退出

📋 日常使用检查清单

  • ✅ 开机后：openclaw gateway start
  • ✅ 遇到问题：openclaw doctor --repair
  • ✅ 配置更改后：openclaw gateway restart
  • ✅ 查看状态：openclaw status --deep
  • ✅ 调试问题：openclaw logs --follow
  • ✅ 更新软件：openclaw update

🔄 重新安装/重置流程

  1. 备份数据： openclaw backup create
  2. 重置配置： openclaw reset --scope config
  3. 重新配置： openclaw onboard
  4. 重启服务： openclaw gateway restart
